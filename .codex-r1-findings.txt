[P1] E19 as proposed will not measure the actual outbox write. emit is fire-and-forget: queue-consumer.ts:196 calls emitOutboxEventOutsideTx(...).catch(...) and returns immediately. Timing around emitTalkEvent only measures promise creation, not appendOutboxEventOutsideTx, notify enqueue, or DO delivery. Instrument inside outbox-emit.ts:76 and streaming-notify.ts:52 instead, keyed by runId/event id.

[P1] The table cannot honestly reconcile to client t2-t1 with only Worker logs. E20 includes DO drain, WebSocket send, and browser receipt, but the proposed probes stop before client arrival. Also do not subtract client Date.now() from Worker Date.now(); clock skew makes that garbage. Use one monotonic server timeline plus a separate client-observed residual, or instrument the DO with event ids.

CF documents a 30-second waitUntil limit shared across all waitUntil calls for a request. Slow/canceled tails can disappear from the sample, biasing attribution toward successful fast runs. Track fallback logs, terminal run status, missing-start events, and waitUntil cutoff symptoms.

[P2] Statement counter design is under-specified and likely wrong. getDbPg() returns the user transaction inside withUserContext, request SQL outside it, and getOutOfBandSql() for streaming emits is a separate path: db.ts:147 and db.ts:171. A one-time Proxy around getDbPg() can miss tx statements, out-of-band streaming inserts, and withUserContext setup statements unless it wraps every SQL function actually placed into ALS.

[P2] withUserContext is not "tx open = 1 stmt". It runs BEGIN plus set local role authenticated and select set_config(...) before user code: db.ts:266. Counting it as one statement will distort E4 and downstream "stmts" attribution.

Also randomize or reset conversation state, because loadTalkContext cost depends on history, attachments, and cached connection state.

Make the instrumentation commit single-purpose, tag its SHA in the plan, verify git diff origin/main -- src/ is empty after revert, and deploy the reverted build before opening the docs-only PR.
