P1 — `activeCount > 0 && primaryCount = 1` is not equivalent to the existing chain.
  Counterexamples:
    (a) 1 active primary row + 1 null-FK non-primary row: snapshot returns healthy,
        existing prune deletes the null-FK row.
    (b) 1 active non-primary row + 1 null-FK primary row: snapshot returns
        activeCount=1, primaryCount=1 → healthy. Existing code deletes the null row,
        then UPDATEs the surviving row to be primary.
  Edge-case claim in plan §4.1 is false.
